# Analysis

!!! info "What you Need to Know"

    Identify the purpose and functional requirements of a problem that relates to the design and implementation of a program.

    Functional requirements are identified in terms of:

    * Inputs
    * Processes
    * Outputs

## What is Analysis?

Analysis is the first stage of software development.

During analysis, the programmer works out:

* what the program needs to do
* what information is needed
* what processing must take place
* what results the program should produce

One of the most important tasks in analysis is identifying the **functional requirements**.

---

# Functional Requirements

Functional requirements describe the tasks a program must perform.

At National 5, functional requirements are usually identified as:

| Requirement Type | Description | Example |
|-----------------|-------------|---------|
| **Input** | Data entered into or collected by the program | Username, age, score |
| **Process** | The work carried out by the program | Calculating an average, sorting data |
| **Output** | Information produced by the program | Messages, reports, results |

A useful way to remember this is:

> **Input → Process → Output**
>
> Data goes in, the computer does something with it, and information comes out.

---

## Inputs

Inputs are the data that enter the program.

Common National 5 examples include:

* keyboard input
* mouse clicks
* touchscreen presses
* sensor readings
* data read from a file

### Examples of Inputs

| Program | Input |
|----------|--------|
| Quiz program | User's answer |
| School registration system | Student ID |
| Weather station | Temperature reading |
| Online shop | Product selected |

---

## Processes

Processes are the actions carried out by the program.

The process is often described as the program's **thinking**.

Examples include:

* calculations
* comparisons
* searching
* sorting
* validation checks
* updating totals

### Examples of Processes

| Program | Process |
|----------|---------|
| Quiz program | Check if answer is correct |
| Banking app | Calculate account balance |
| Online shop | Calculate total cost |
| School system | Search for a pupil record |

---

## Outputs

Outputs are the results produced by the program.

Common outputs include:

* text displayed on screen
* images or graphics
* sounds
* printed documents
* saved files

### Examples of Outputs

| Program | Output |
|----------|--------|
| Quiz program | Final score |
| Banking app | Current balance |
| GPS system | Route displayed on a map |
| Online shop | Order confirmation |

---

# Worked Example 1

## Scenario

> A program asks every pupil in a class for their age. The program then calculates the average age and displays the result.

### Step 1: Identify the Inputs

* Ages entered by the pupils

### Step 2: Identify the Process

* Calculate the average age

### Step 3: Identify the Output

* Average age displayed on the screen

| Inputs | Processes | Outputs |
|----------|----------|----------|
| Ages | Calculate average age | Average age |

---

# Worked Example 2

## Scenario

> A GPS navigation system asks the user for a destination. The system determines the current location, calculates a route and displays directions on a map.

| Inputs | Processes | Outputs |
|----------|----------|----------|
| Destination entered by the user | Calculate the best route | Route shown on map |
| Current location from GPS satellite | Calculate journey time | Estimated arrival time |

---

# National 5 Exam Tip

When an exam question asks for inputs, processes and outputs:

1. Look for the data entering the system.
2. Look for the work being carried out.
3. Look for the information being produced.

A good answer normally uses words taken directly from the scenario.
