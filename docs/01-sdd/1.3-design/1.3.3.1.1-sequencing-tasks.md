# Sequencing Tasks

These tasks will help you practise showing sequencing using different types of design notation.

Use the same problem in each section, but represent it using a different notation.

!!! info "Draw.io"

    Use [draw.io](https://app.diagrams.net) to create your structure diagrams and flowcharts.

    Export each diagram as an image and add it to your workbook.

## Structure Diagrams

For each task, create a __structure diagram__.

Remember:

* the main task goes at the top
* sub-tasks go underneath
* sequencing is read from left to right
* each task should happen once, in order

___

### Task 1: Total Wage

Create a structure diagram for a program that:

1. asks the user for their hourly pay rate
2. asks the user how many hours they have worked
3. multiplies the hours worked by the hourly rate to calculate the total wage
4. displays the total wage

Add an image of your structure diagram to your workbook.

___

### Task 2: Area of a Circle

Create a structure diagram for a program that:

1. asks the user for the radius of a circle
2. calculates the area using `2 * π * radius^2`
3. displays the area

Add an image of your structure diagram to your workbook.

___

### Task 3: Volume

Create a structure diagram for a program that:

1. asks the user for the length
2. asks the user for the breadth
3. asks the user for the height
4. calculates the volume
5. displays the volume

Add an image of your structure diagram to your workbook.

___

### Task 4: Running Total

Create a structure diagram for a program that:

1. asks the user to enter a number
2. adds the number to a total
3. repeats this until five numbers have been entered
4. displays the total

Add an image of your structure diagram to your workbook.

    This task includes repetition, so it is a challenge question. Focus on showing the tasks in the correct order.

## Pseudocode

For each task, write __design pseudocode__.

Use this format:

```pseudocode
Main Steps

1 ...
2 ...
3 ...

Refinements

1.1 ...
2.1 ...
3.1 ...
```
