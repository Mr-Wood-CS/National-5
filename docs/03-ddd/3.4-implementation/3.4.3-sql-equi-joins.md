# SQL Equi-Join Operations

!!! info "What You Need to Know"

    For National 5 Computing Science, you should be able to:

    - Use an equi-join to retrieve data from two related tables.
    - Identify the matching fields used to join two tables.
    - Understand that an equi-join uses a `WHERE` clause.
    - Explain how records are matched using equal values in both tables.
    - Recognise primary key and foreign key fields in a join.

---

## What Is An Equi-Join?

An equi-join is used to display data from two related tables in one query.

The tables are joined using fields that contain matching values.

For example, a `Pupils` table might store pupil details, and a `Guidance` table might store guidance teacher details.

If both tables contain a `House` field, SQL can use that field to match the correct pupils with the correct guidance teacher.

At National 5, an equi-join is written using:

- `SELECT`
- `FROM`
- `WHERE`

The `WHERE` clause is used to show which fields must match.

---

## Before The SQL

Before writing an equi-join, it helps to remember the database words.

| Word | Meaning |
|------|---------|
| Entity | A table in the database |
| Attribute | A field or column in a table |
| Record | One row of data in a table |
| Primary key | A field that uniquely identifies a record |
| Foreign key | A field that links to a primary key in another table |

!!! example "Example"

    In a `Pupils` table:

    * the entity is `Pupils`
    * `Forename`, `Surname` and `House` are attributes
    * one pupil's details make one record

When joining tables, look for an attribute that appears in both tables or links the two tables together.

This is the matching field.

---

## Equi-Join Syntax

```sql title="Equi-Join Syntax" linenums="1"
SELECT *
FROM entity1, entity2
WHERE entity1.column1 = entity2.column1;
```

In this syntax:

- `entity1` is the first table.
- `entity2` is the second table.
- `column1` is the matching field.
- The field on the left of the equals sign comes from one table.
- The field on the right of the equals sign comes from the other table.

Only records with matching values will be displayed.

## Example Tables

The `Guidance` table stores information about guidance teachers.

==Entity = Guidance==

| EmployeeNum | Title | Surname | House |
|-------------|-------|---------|-------|
| 1 | Mr | Stewart | Nevis |
| 2 | Miss | Kelly | Lochiel |
| 3 | Mr | Macdonald | Rannoch |
| 4 | Mrs | Jordan | Shiel |

The `House` field is an attribute in the `Guidance` table.

The `Pupils` table also contains a `House` attribute.

==Entity = Pupils==

| StudentNum | Forename | Surname | House |
|------------|----------|---------|-------|
| 1 | Amy | Smith | Nevis |
| 2 | Ben | Campbell | Lochiel |
| 3 | Cara | Fraser | Rannoch |
| 4 | Daniel | Brown | Nevis |

The two tables can be linked because both entities store house names.

- One guidance teacher is responsible for many pupils.
- Each pupil belongs to one house.
- The `House` attribute contains matching values in both tables.

!!! tip "How To Spot The Matching Field"

    Ask yourself:

    * Which attribute appears in both tables?
    * Which values are repeated in both tables?
    * Which field links the records together?

## Equi-Join Example

The following query displays pupil and guidance information together.

```sql title="Join Pupils And Guidance" linenums="1"
SELECT *
FROM Pupils, Guidance
WHERE Pupils.House = Guidance.House;
```

This query selects all fields from both tables.

The most important line is the condition:

```sql
Pupils.House = Guidance.House
```

matches records where the pupil's house is the same as the guidance teacher's house.

For example, a pupil in `Nevis` would be matched with Mr Stewart because both records contain the value `Nevis` in the `House` field.

---

## Reading The Query Step By Step

| SQL Line | What It Does |
|----------|--------------|
| `SELECT *` | Display all attributes from the joined records |
| `FROM Pupils, Guidance` | Use the `Pupils` and `Guidance` entities |
| `WHERE Pupils.House = Guidance.House` | Match records where the `House` values are equal |

!!! info "Key Idea"

    The equals sign in an equi-join does not mean "store this value".

    It means "only show records where these two values match".

## Selecting Specific Fields

You do not always need to display every field.

You can choose only the fields needed in the output.

```sql title="Select Specific Fields" linenums="1"
SELECT Pupils.Forename, Pupils.Surname, Guidance.Title, Guidance.Surname
FROM Pupils, Guidance
WHERE Pupils.House = Guidance.House;
```

This would display:

- the pupil's forename
- the pupil's surname
- the guidance teacher's title
- the guidance teacher's surname

Using the table name before the attribute name makes it clear which table the attribute comes from.

!!! tip "Exam Tip"

    When identifying an equi-join, look for a `WHERE` condition that compares a field from one table with a matching field from another table.

    ```sql
    WHERE Pupils.House = Guidance.House;
    ```

## Summary

| SQL Feature | Purpose |
|------------|---------|
| `SELECT` | Chooses the fields to display |
| `FROM` | Names the tables being used |
| `WHERE` | Gives the matching condition |
| `=` | Matches equal values in both tables |
| Entity | A table used in the query |
| Attribute | A field used in the query |
| Equi-join | Combines records from related tables |

Equi-joins are used when information is stored across more than one related table.
