## What Is A Data Dictionary?

A data dictionary is created during the design stage of database development.

It describes how each table in a database should be built.

You can think of it as a blueprint for the database.

It tells the developer:

- what fields are needed
- what type of data each field will store
- how much data can be stored
- which fields are keys
- what validation should be used

!!! example "Simple Example"

    | Field Name | Data Type | Size | Key | Validation |
    |------------|-----------|------|-----|------------|
    | Student_ID | Number | 6 | PK | Presence check |
    | First_Name | Text | 20 | | Presence check |
    | Class_ID | Number | 3 | FK | Presence check |

---

## Why Is A Data Dictionary Useful?

A data dictionary helps make sure the database is planned properly before it is created.

It helps to:

- keep data consistent
- reduce mistakes
- make field names clear
- choose sensible data types
- show how tables are linked
- help different developers understand the same design

Without a data dictionary, developers may store the same data in different ways.

For example, one developer might use `PupilID`, while another might use `Student_Number`. A data dictionary avoids this kind of confusion.

## Summary

A data dictionary records the design of a database table.

It should include:

- field names
- data types
- field sizes
- primary and foreign keys
- validation rules
- clear reasons for each design choice
