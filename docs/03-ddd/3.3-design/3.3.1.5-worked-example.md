# Scenario

A school wants to store details about pupils.

The database needs to store:

- a unique pupil ID
- the pupil's forename
- the pupil's surname
- the pupil's year group
- the pupil's house
- whether the pupil receives free school meals

---

## Completed Data Dictionary

| Field Name | Data Type | Size | Key | Validation |
|------------|-----------|------|-----|------------|
| Student_ID | Number | 6 | PK | Presence check |
| Forename | Text | 20 | | Presence check |
| Surname | Text | 30 | | Presence check |
| Year_Group | Number | 2 | | Range check: 1 to 6 |
| House | Text | 10 | | Restricted choice |
| Free_Meals | Boolean | | | Restricted choice |

---

## Explaining The Choices

`Student_ID` is a primary key because it uniquely identifies each pupil.

`Forename` and `Surname` are text because they store names.

`Year_Group` is a number because it stores a year group value.

`House` is text because it stores a house name.

`Free_Meals` is Boolean because it stores either Yes or No.

---

## Exam Tip

When justifying a data dictionary, do not just name the data type.

Explain why it is suitable.

| Weak Answer | Better Answer |
|-------------|---------------|
| `Surname` is text. | `Surname` is text because it stores letters and will not be used in calculations. |
| `Age` is number. | `Age` is number because it stores a whole number and can be compared using range checks. |
| `Free_Meals` is Boolean. | `Free_Meals` is Boolean because it stores only two possible values: Yes or No. |
