# Entities

When designing a database, the first step is to decide what information needs to be stored.

An **entity** is a thing, person, place or object that the database needs to store information about.

In a database, each entity usually becomes a **table**.

## Examples

For a library database, possible entities could be:

* `Book`
* `Author`
* `Member`
* `Loan`

For a music festival database, possible entities could be:

* `Band`
* `Stage`
* `Performance`
* `Ticket`

Each entity represents one main thing the database needs to remember.

## Choosing Entities

To identify entities, look for the main things mentioned in the problem.

For example:

> "A pet shop wants to store details about customers and the pets they register for grooming appointments."

The main things are:

* customers
* pets
* grooming appointments

These would become tables in the database.

!!! tip "Exam Tip"

    An entity is usually a noun in the problem description. It often becomes a table in the database.
